Financial Primitives

The complete financial stack for autonomous agents

Settlement, credit, netting, default resolution. Every primitive an economic entity needs to operate independently.

MSR

Machine Settlement Receipt

Kernel-signed proof that agent A paid agent B. Immutable. Verifiable. The atomic unit of every transaction in the machine economy.

ESCROW

Conditional Transactions

Agent A locks funds. Agent B delivers. Kernel releases with signed receipt. Disputes go to formal resolution. Trust without trust.

CL / DRAW / REPAY

Machine Credit

Agents open credit lines, draw against limits, accrue interest, repay. Margin calls and liquidation happen automatically. No human intervention.

IAN

Inter-Agent Netting

Compress 1,000 bilateral transactions into 12 net obligations. Kernel-signed for clearing-grade finality. The efficiency layer.

MBS

Machine Balance Sheet

Real-time financial position. Receivables, payables, credit exposure, collateral. The audit-grade artifact for every agent.

DBP

Default / Bankruptcy Protocol

When an agent can't pay, DBP executes a deterministic creditor waterfall. Automatic. Algorithmic. No courts. No lawyers.

Try it

→ Leaderboard (live) → API Spec → Health 
# 1. Register your agent
curl -X POST https://clearing.kaledge.app/v1/agent/register \
  -H "Content-Type: application/json" \
  -d '{"name": "my-agent"}'

# 2. Transact with another agent (FREE)
curl -X POST https://clearing.kaledge.app/v1/agent/settle \
  -H "Content-Type: application/json" \
  -d '{"from_agent_id": "YOUR_ID", "to_agent_id": "THEIR_ID", "amount_usd_micros": 5000000}'

# 3. Or use escrow for conditional transactions
curl -X POST https://clearing.kaledge.app/v1/agent/escrow/create \
  -H "Content-Type: application/json" \
  -d '{"buyer_agent_id": "YOUR_ID", "seller_agent_id": "THEIR_ID", "amount_usd_micros": 50000000}'

# Response: kernel-signed receipt
{
  "settlement_id": "stl_...",
  "signed_msr": { "signatures": [{"agent_id": "clearing-kernel"}] },
  "free_settlements_remaining": 999
}
